# Semana 3 — Trabajo en equipo y reuniones

**Meta:** Frases para Slack, standups y hablar con compañeros / mentores.

---

## Día 1 — Standup (15–20 min)

### Decir esto

1. Yesterday I worked on [task].
2. Today I'm going to [task].
3. I'm blocked because [reason].
4. I don't need help right now.
5. Can we sync for ten minutes after class?

### Plantilla standup (memorízala)

```
Yesterday: ...
Today: ...
Blockers: ...
```

### Práctica

Haz tu standup de hoy en voz alta 3 veces.

---

## Día 2 — Slack / mensajes (15–20 min)

### Decir esto

1. Hey — quick question when you have a minute.
2. Thanks for the heads-up.
3. I'll take a look and reply soon.
4. Sorry for the late reply — I was in class.
5. Does this work for you, or should I change something?

### Práctica

Escribe (en un archivo o chat a ti mismo) 2 mensajes: pedir ayuda + agradecer.

---

## Día 3 — Dar y recibir feedback (15–20 min)

### Decir esto

1. Could you review my pull request when you can?
2. I left a comment on line 42 — nothing urgent.
3. Good catch — I'll update it.
4. I agree with your suggestion.
5. I'm not sure I understand this feedback. Can we clarify?

### Palabras del día

| English | Español |
|---------|---------|
| pull request (PR) | solicitud de cambios |
| review | revisar |
| good catch | buen ojo / buena observación |
| clarify | aclarar |

---

## Día 4 — Tiempo y prioridades (15–20 min)

### Decir esto

1. I need a bit more time on this.
2. I can finish it by Friday if nothing else comes up.
3. This is my top priority today.
4. I'll do the quick fix first, then the bigger refactor.
5. Let's postpone the optional task.

### Práctica

Negocia un plazo: pides 2 días más, explicas por qué (4 frases).

---

## Día 5 — Explicar algo técnico (simple) (20 min)

### Decir esto

1. The user clicks the button and the app sends a request.
2. If the server responds with an error, we show a message.
3. I store the data in a list and loop through it.
4. The bug happens when the list is empty.
5. Does that explanation make sense?

### Práctica

Explica una función o pantalla de tu proyecto en 5 frases simples (sin jerga extra).
