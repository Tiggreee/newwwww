# Ilustraciones bien referenciadas (ruta Data)

Objetivo: usar imagenes que ayuden al aprendizaje, se sientan cercanas para alumnos de Data y cumplan licencia para evitar problemas legales.

---

## 1) Fuentes recomendadas (seguras para uso educativo)

Usa solo bancos con licencia clara y guarda siempre URL + autor + fecha de consulta.

1. Unsplash
- Licencia: Unsplash License (uso amplio, revisar restricciones de redistribucion masiva).
- Que usar aqui: fotos de oficina, estudio, laptops, pizarras, trabajo en equipo.

2. Pexels
- Licencia: Pexels License (uso amplio, revisar terminos vigentes).
- Que usar aqui: escenas de estudio, grupos, videollamadas, presentaciones.

3. unDraw
- Licencia: libre para proyectos (confirmar terminos actuales en su sitio).
- Que usar aqui: ilustraciones limpias de tecnologia, aprendizaje, colaboracion.

4. Storyset (Freepik)
- Licencia: segun plan y atribucion requerida en algunos casos.
- Que usar aqui: personajes y escenas didacticas para explicar procesos.

5. Wikimedia Commons
- Licencias mixtas (CC BY, CC BY-SA, dominio publico, etc.).
- Que usar aqui: diagramas base (estadistica, ciencia de datos, visualizacion).

Nota: nunca asumas licencia por apariencia. Si no hay licencia explicita, no se usa.

## 2) Formato obligatorio de referencia

Debajo de cada ilustracion agrega una linea de credito.

Plantilla corta:
- Fuente: [Plataforma], Autor: [Nombre], Licencia: [Tipo], URL: [Enlace], Consulta: [AAAA-MM-DD]

Plantilla para README o materiales del curso:
- "Imagen de [tema]. Fuente: [Plataforma] por [Autor] ([Licencia]). Disponible en: [URL]. Consultado el [fecha]."

Ejemplo real (estructura):
- Fuente: Unsplash, Autor: Jane Doe, Licencia: Unsplash License, URL: https://unsplash.com/photos/abc123, Consulta: 2026-05-24

## 3) Estilo visual para que se sienta familiar

Para alumnos de ruta Data, prioriza escenas que conecten con su dia a dia:

1. Contextos
- Dashboard con graficas.
- Persona revisando SQL en laptop.
- Reunion breve tipo standup.
- Presentacion de hallazgos con 2-3 graficas.

2. Tonalidad
- Colores claros y amigables.
- Evitar imagenes demasiado corporativas o frias.
- Incluir diversidad de perfiles (edad, genero, contextos).

3. Regla didactica
- Una imagen = una idea.
- No decorar por decorar.
- Si no mejora comprension, no se agrega.

## 4) Ejemplos de ilustracion por tema (Oracle Data path)

1. SQL basico (SELECT, WHERE, GROUP BY)
- Ilustracion sugerida: persona comparando dos tablas en pantalla.
- Texto de apoyo en clase: "Filter rows first, then group results."

2. Joins
- Ilustracion sugerida: dos conjuntos que se intersectan + tabla resultado.
- Texto de apoyo: "INNER JOIN returns only matching records."

3. Calidad de datos
- Ilustracion sugerida: checklist con duplicados, nulos y formatos.
- Texto de apoyo: "Clean data before analysis."

4. Visualizacion
- Ilustracion sugerida: grafica de barras vs lineas para distintos casos.
- Texto de apoyo: "Choose the chart based on your question."

5. Storytelling
- Ilustracion sugerida: persona explicando 3 insights en una slide.
- Texto de apoyo: "Insight, impact, next action."

---

## 5) Checklist legal antes de publicar

- La fuente muestra licencia explicita.
- Se guardo URL original.
- Se registro autor.
- Se anoto fecha de consulta.
- La licencia permite uso educativo/publicacion.
- La atribucion esta visible en el material.

Si falla un punto, reemplaza la imagen.

---

## 6) Recomendacion practica para este repo

Crea una carpeta local de apoyo (fuera del repo publico si quieres):
- assets-referencias/
  - imagen-01.jpg
  - creditos.md

En creditos.md guarda por cada imagen:
- Nombre local del archivo
- Tema del modulo
- Autor
- Plataforma
- Licencia
- URL
- Fecha de consulta

Asi todo queda trazable si luego haces presentaciones, PDF o contenido web.
